Custom Blocks management

Create and manage the building blocks of your website layout, such as business information, advertisements, contact forms, images or media embeds from other platforms.

There are two main types of blocks:

  • Custom blocks are created by selecting a custom type : Auto-Format (BBCode, emoticons), PHP or HTML.
  • Modules blocks are pre-built blocks available when you install and enable a module, such as "User Login" or "Main Menu".

Block Settings

Each block has the following default properties :

  • Name ( Title )
  • Order, or weight, of display
  • Side area ( refer to theme layout )
  • Enabled or disable status
  • Target front-page and module(s)
  • User groups permissions
  • Independent Cache settings
  • Independent options ( edit )
  • Custom Blocks Options
  • + Type of Block (Auto-Format, HTML, PHP)
  • + Copy/duplicate
  • + Edit the template
  • + Delete the custom block

Block Side

The block "side"" is just a legacy code convention for identifying areas of a Theme and organizing the layout. Blocks are modular units of content that can be positioned almost anywhere on the page. The Theme layout defines the "side" areas available.

For example, a theme can place the block-left "Main Menu" at the top of the page to create a navigation bar.

A Theme defines the overall structure, positioning areas and appearance. While templates of Module and Blocks specify the shape. The theme and templates can be replaced with a custom design.

Types of blocks

Custom blocks

A Custom block is created with a custom type : Auto-Format (BBCode, emoticones), PHP, or HTML, CSS and JavaScript. Thus allowing you to design and easily manage reusable Components for an effective website layout.

BBCode is a simple and secure way to format your entries (like bold text, italic text, links). Nowadays, Markdown shares the same purpose as BBCode.

Custom Blocks can also embed code or integrate with third-party features specific to a site's unique needs.

The block editor will switch according to the type of content.

Modules blocks

Modules have several pre-built blocks that offer specific functionality and favor composition over inheritance. These blocks are made available in Block Management by enabling modules. Each module block has its own settings and customization options.

Templates

A front-end theme defines the overall look. While module and block templates shape the content.

A custom template can be created with PHP or HTML, CSS and JavaScript to be used as a single file component.

Template Set

When modules are installed, their templates become part of the Default Template Set.
These templates can be easily edited or duplicated with the module Components (AltSys).
The Template Sets available can be found in Template Set management of module Render.

1. First, make a copy of the Default Template Set ( Base-File )

2. Browse to templates ¬ª¬ª Module

3. Click "edit" next to the template of the new Set

4. Make the changes you want

5. Save the template

In case of errors, check the box next to the template to easily copy the file from the Base-Default Set !

Find a Template

Template management allows templates to be filtered by module, sorted and searched by name.

Naming Convention

The template naming convention is a systematic method for naming files written in all-lowercase ASCII letters. The name specs might include the package name, action, event. For example:

  • [module-name]_[admin].html
  • [module-name]_[block].html
  • [module-name]_[inc]_[component-name].html
