Overview

The utility module Components (Alternative System Library, AltSys) provides quick access to common tasks and general management of custom components, blocks, templates, language internationalization (i18n) and localization (l10n).

  • Advanced Utilities to customize templates

Edit a Custom Block

You can edit Blocks from the modules Components and System »» Block Management.

If your theme includes the kebab menu, you can edit Blocks from the front end.

The Block Title kebab menu (three vertical dots)

The kebab menu, also known as the vertical three-dot menu, is an icon used to open a menu and display actions for the related item : quick links to block management.

Delete a Custom Block

Click the Delete icon in the upper right corner of the block's card.

In the confirmation page that is displayed, confirm to permanently remove the Custom Block from your database.

Modules Blocks

Modules can provide several unique blocks. Each module block has its own parameters and template. When you install a module, only one instance of each block is available in Block Management. "Components" provides an additional functionality, "Force Clone" allows you to duplicate any available block.

Cloning Modules Blocks

Perform the following steps to clone a block.

  1. Click the module with the block you want to duplicate
  2. Click the button "force clone" in the block's card
  3. Enter a new block title and edit settings
This procedure creates an exact copy of the block with the new title and settings.

Create a Custom Template

  1. Create a custom template
  2. Add some content
  3. Name the template e.g. : custom_template.html *
  4. Save the custom template
* The name of the Custom Template cannot be changed once it has been saved in the database !

Using your Custom Block

You can include your custom template in your theme layout, modules templates or custom blocks.

Modules Templates

The Render engine provides the additional functionality to store and manage templates in the database. Using the database and caching compiled templates makes customization easier and greatly improves performance.

When modules are installed, their template files are added to the default template set (DB-DEFAULT), which is stored in the database. Thanks to the flexibility of the rendering engine, you can duplicate an existing template set and adapt the predefined templates to the needs of your organization and your users.

Language Localisation

The built-in language manager aims to facilitate the localization process and branding customization. It leverages the adaptability of modular architecture to align UI elements with a brand's tone and voice.

Language localisation is the process of adapting a product's translation to a specific country or region. It is the second phase of a larger process of product translation and cultural adaptation to account for differences in distinct markets, a process known as internationalisation and localisation.

Language and character set

UTF-8 Unicode is the common character encoding method and the default character set for distributed languages, including english, fr_utf8, ja_utf8, pt_utf8, ru_utf8.

Edit language translation

Language Placeholders

Language constants can contain technical data like HTML markup and placeholders. These elements should not be modified or translated.

It's important to keep any placeholder defined within a constant, for example, %s or {0}

The purpose of the placeholder is to temporarily take the place where a value is dynamically inserted.

For example, define( '_MSC_INTSITE' , 'Interesting Site: %s'); here %s is a placeholder for your site name.

You can add new languages string constants to be used in your theme components or custom templates.
