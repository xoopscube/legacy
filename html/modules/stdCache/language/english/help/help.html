Standard Cache Management

This module utility improves the performance of your web application by allowing users to retrieve faster compiled temporary files stored in a cache directory.

The module manages the standard cache policy of your web application platform cache files for every user group except the webmaster group. To ensure the best performance, it also provides a block to easily clear the cache.

Features & Process Flow

The Standard Cache module provides robust tools to manage your website's cache, enhancing performance and stability. Key features include:

  • Comprehensive Cache Management: Monitors and manages various cache types including Smarty cache, compiled templates, system logs, and user uploads.
  • Configurable Limits: Administrators can set specific size limits for different cache types, including overall limits, notification thresholds, and automated cleanup thresholds.
  • Automated Admin Notifications: When a cache directory approaches or exceeds its configured notification limit, an email alert is automatically sent to administrators.
  • Intelligent Email Delay: After a notification email is sent, a record of the notification time (last_cache_alert_time) is stored. This prevents repeated emails for the same issue until a defined interval (e.g., 24 hours) has passed, avoiding alert fatigue.
  • Automated Cache Cleanup: If cache sizes exceed their configured cleanup limits, the module can automatically clear older or less-used files to free up space.
  • Manual Cache Clearing: Provides an admin interface to manually clear specific cache types, with options to clear all files or only those older than a certain age.
  • Cache Statistics: Offers a dashboard view of current cache sizes, file counts, and configured limits, helping administrators monitor cache health.

Process Flow Summary:

  1. Configuration: Admins set cache size limits (general, notification, cleanup) and enable/disable notifications in the module preferences.
  2. Monitoring & Notification: The system continuously monitors cache usage. If a cache size exceeds its *notification limit*, an email alert is triggered.
  3. Notification Cooldown: The last_cache_alert_time is updated to ensure emails are not sent too frequently (e.g., respecting a 24-hour delay).
  4. Automated Cleanup: If a cache size exceeds its *cleanup limit*, the module automatically removes older files to bring usage back within acceptable levels.

This combination of monitoring, notification, and cleanup helps maintain optimal server performance and resource usage.

Understanding Group Policy Caching

For security reasons, the Standard Cache Policy should prevent the caching of files for administrators' access. Otherwise, the files cached could be accessed by anonymous users.

Also, to prevent the cache to grow larger with too many files in your server, the module cache can be enabled or disabled from the general settings :

Best practices for cache control settings

Each module and block allows you to set how long the module or block should keep cached files.

Therefore, the standard cache policy can cache a significant number of files.

For this reason, the module provides a block with the ability to set a time interval for clearing cache files.

Clear cache block

To ensure the best performance in case of increased demand or scheduled downtime for maintenance of your server, it is recommended to install the block "Clear cache block".

Install the module's block to clear precompiled files to cache.

  1. Install the block "Clear cache block"
  2. Select the target modules (multiple selection)
  3. Select the target groups (multiple selection)
  4. Enter the interval timer (minutes). Default value : 60
This block is not visible and can delete all of cached files, including extra cache files, with interval timer.
