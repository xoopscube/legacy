How to get help

There are a few different ways to find help for your Web Application Platform. You can use the Action Search to assist you in searching modules Help files and preferences. You can learn more about a module from the contextual Help dialog and the built-in User Guide.

Using Action Search

Click the x-cube on the right to open the admin panel. Enter different keywords in the action search field to find what you are looking for in the modules User Guide files and modules Preferences.

>Using Built-in Help

All modules installed on your Web Application Platform have a built-in User Guide. On the left sidebar, for each module, there is a help link, so you can easily get the User Guide for the modules you are using. A module's User Guide can provide easy access to other sections, topics, and examples.

Using Contextual Help Dialog

The Help button is available in the administration of most modules. This option displays a modal dialog box to get the corresponding help section. For example, this button displays the contextual help for module managent.

Block Management

Block Management allows you to administer and maintain blocks of installed modules and custom blocks that you create. Simply put, there are two types of blocks: modules blocks and custom blocks.

Modules Blocks

Modules blocks are pre-built blocks such as "User Login", "User Menu" or "Main Menu". Each module block provides specific functionality and has its own settings and template.

Custom Blocks

Custom blocks are a quick and easy way to extend and customize your content with images, text or third party services: maps, videos, etc. By default, Custom blocks are rendered with a dummy template. However, it is possible to create reusable components with their own custom template. See more details in the section custom and reusable components.

Block Actions

Frontend themes usually include a component with block actions. Webmasters can access the menu next to the Block Title to manage a block.

It is noteworthy that all blocks can also be managed in the "Components" module, although the user interface is slightly different.

Module Management

The functions necessary for the configuration and management of your Web Application are grouped into eight required modules : Components, System, Render, User Groups, Profile, Cache, Message and Protector.

The recommended modules are not mandatory, but they extend the application's functionality and content editing features : CKEditor, D3Forum, Pico and X-Update Manager.

Active modules are represented by the icons in the vertical sidebar of the Control Panel.

Modules mostly work independently of one another. However, in a number of cases the functionality of certain modules depends on the capabilities of other modules. For example, the module Pico relies on the following dependencies : Components (altsys library) to manage blocks, templates and language, and D3Forum for comments-integration.

Module Installation

You can easily get and deploy compatible modules with X-Update manager. Otherwise, you can install modules manually:

  • Download a compressed archive, extract and upload to your server using FTP
  • Upload a compressed archive and extract files with your hosting file manager
The files and folders of the compressed archive are based on the directory structure of the current version.

Click "install module" to get a list of available modules.

  1. Click the "Install" button of the module card.
  2. Follow the steps in the installation wizard.
  3. Configure the module's preferences
  4. Edit module order, cache and User Groups permissions

Usually the module instructions are available in the contextual help of each page, and you can access the documentation at any time from the admin sidebar. Otherwise, refer to the module's readme.txt file for specifications.

Module Active

You can activate or deactivate a module or several modules in bulk by using the switch on the card of each module.
