Getting Started

A kick-start guide to XCL if you are running this admin dashboard for the first time.

The goal of this kick-start guide is to help new site administrators get orientated to what this position does and what responsibilities it entails. It provides the minimum amount of information needed to get started and begin with your Web Application Platform and CMS.

Other detailed instructional materials are available and will be highlighted throughout.

Quick Links (Top)

From quick links, you can access and search the most relevant features.

If you have installed Xupdate a top notification will show you the available updates.

There is also a link to the recommended file manager where you can upload files to the server without the need for additional FTP access.

Left Panel

The menu on the left gives you access to an overview of the structure and the features that you have at your disposal to create and manage your web application platform.

In short, you track notifications and manage the structure of your website from the dashboard. Navigate in the easiest way between the core and third-party modules.

Right Panel (the cube)

The right panel, also called "the cube", allows you to switch between instances or multi-sites, if the service is installed and running on your web platform.

For example, you can navigate have multi-wordpress switch to an instance to manage pages, posts, etc.

This service will also have a button to quickly add individual sections, i.e. tasks, contacts.

Main container

Depending on whether a module, category, or content is selected, different content is loaded in the main container.

The main container will show a breadcrumb menu to make your navigation easier.

X-Update Store

If the module is installed, you have free access to the public store of trusted developers.

You can install add-ons, modules, themes and Preloads (Single File Component).

Settings

The Settings section is the place where you can customize your website.

From there you are able to close the site to public but allowing access to specifc groups.

Set the name, language, default theme, landing module, user account, email, and much more.

Blocks management

The block management system is built into the kernel and provides five default block positions, the order of display, in which module, for which user groups and cache settings.

The block position is just a legacy code convention to identify areas in the theme layout.

The block template specifies the shape, and the global theme specifies the areas and appearance. However, it can be overridden by a custom template.

There are two types of blocks : Modules blocks and Custom blocks.

Modules blocks

Modules have several pre-built blocks that offer specific functionality and favor composition over inheritance. These blocks are made available in the block management by enabling modules.

Custom blocks

A Custom block can be built with HTML, CSS, JavaScript, PHP and Smarty variables. Thus allowing to create reusable Components to create an effective website layout.
It can also embed code or integrate with third-party features specific to a site's unique needs.

The block editor will switch according to the type of content.

Comment Management

The block management system is built into the kernel. It provides a text comment box where users can generate their own comments and leave them on a specific piece of content.

You can follow the last comments of your articles on your Dashboard. In addition to having a quick view, you can also give Reply to the comment as well as moderate its status as a public, unpublish, spam, and delete it.
D3 modules comments are easily integrated (module preferences) and managed through D3Forum.

Category management

Here you can create and edit categories, set the maximum file size, maximum image width and height, display and hide categories and images.

The following options are available when creating a category:

Name

A descriptive name of the category.

Permissions

  • Groups who can upload to the category.
  • Groups who can view the category.

File size

You can optionally set a maximum allowed size in kilobytes for uploaded file. Leave blank if you do not wish to impose a limit.

Example: 1024000 bytes = 1,000Kb = 1MB

Note: Hosting services usually have limits on the amount of data that can be uploaded via PHP If you need to upload very large files, you should contact your server's support service and tell them about the size limits they have put in place. Alternatively, you can install the module Xelfinder, the web file manager can connect to a cloud service.

Image width

Optionally set the maximum allowed width in pixes for uploaded images. Images over this setting will be automatically resized.

Image height

Optionally set the maximum allowed height in pixes for uploaded images. Images over this setting will be automatically resized

Display

Show or hide the category

Order

In the list component, you can sort by any column. Leave the value by default for the original sort order, or set a custom order with a numeric value.

Category store type

Choose either the file or the database storage scheme for your image files.

This parameter cannot be changed later!

Keep in mind that if you select the database as the storage destination, image files will be stored as BLOB type data which may not be suitable for backup purposes.

Word Censor

Control Panel > Preferences > Word Censorship

This section of the Control Panel allows you to enable word censoring. It is also where you define the list of censored words.

Enable censorship?

You may enable or disable word censoring. If you select ‚ÄúYes‚Äù, the system will replace any specified words in channel entries, comments, forum posts, etc. according to your preference below.

Replacement characters

You may optionally specify a word or phrase to be used when replacing censored words. For example, if you set ‚Äútisk tisk‚Äù as your replacement word, and ‚Äúshucks‚Äù is in your censored list, then anytime ‚Äúshucks‚Äù is used it will be replaced with ‚Äútisk tisk‚Äù. If you do not set this preference, a pound symbol will be used for each character that is censored, so ‚Äúshucks‚Äù would be converted to ‚Äú######‚Äú.

Words to censor

Specify a list of words to censor. Wildcards are allowed. For example, test* would censor the words ‚Äútest‚Äù, ‚Äútesting‚Äù, and ‚Äútester‚Äù, while *gress would censor the words ‚Äúprogress‚Äù and ‚Äúcongress‚Äù.

The profanity filter, sometimes referred to as just "filter" or "censor", is a feature typically used on forums and comments that automatically scans users' posts as they are submitted and automatically changes or censors particular words.

Enter the censored words in the textarea using a common delimiter character "|" (pipe) to separate the values.

Set the characters replacing all the censored words in the input field "censor-replace".
