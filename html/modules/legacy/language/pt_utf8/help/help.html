Admin Theme Options

Show or hide blocks and modules Tips

Modules Tips are small alert messages that display short information about functionality.

Actions Tips display descriptive text when you rest the pointer on a button command or control.

Module Help enhanced Help are larger modals that display more descriptive text than a Module Tip and can have a link to a Help article.

You can set an option to show or hide Tips.

Show or hide Tips

  1. Click the x-cube admin panel
  2. Click theme options.
  3. Click help tips.
  4. Under User interface options, in the Admin Theme Options list, click the option that you want:

Show feature descriptions in Tips This option turns on Tips and Enhanced Help-Tips so that you see more information about a module feature, including the functionality name, keyboard shortcuts, and links to Help articles. This is the default setting.

Don't show feature descriptions in Tips This option turns off Enhanced Help Tips so that you see only the brief information and maybe a keyboard shortcut.

Don't show Tips This option turns off Tips and Enhanced Help Tips so that you only see the module's view and action controls.

Blocks management

The block management system is built into the kernel and provides five default block positions, the order of display, in which module, for which user groups and cache settings.

The block position is just a legacy code convention to identify areas in the theme layout.

The block template specifies the shape, and the global theme specifies the areas and appearance. However, it can be overridden by a custom template.

There are two types of blocks : Modules blocks and Custom blocks.

Modules blocks

Modules have several pre-built blocks that offer specific functionality and favor composition over inheritance. These blocks are made available in the block management by enabling modules.

Custom blocks

A Custom block can be built with HTML, CSS, JavaScript, PHP and Smarty variables. Thus allowing to create reusable Components to create an effective website layout.
It can also embed code or integrate with third-party features specific to a site's unique needs.

The block editor will switch according to the type of content.

Category management

Here you can create and edit categories, set the maximum file size, maximum image width and height, display and hide categories and images.

The following options are available when creating a category:

Name

A descriptive name of the category.

Permissions

  • Groups who can upload to the category.
  • Groups who can view the category.

File size

You can optionally set a maximum allowed size in kilobytes for uploaded file. Leave blank if you do not wish to impose a limit.

Example: 1024000 bytes = 1,000Kb = 1MB

Note: Hosting services usually have limits on the amount of data that can be uploaded via PHP If you need to upload very large files, you should contact your server's support service and tell them about the size limits they have put in place. Alternatively, you can install the module Xelfinder, the web file manager can connect to a cloud service.

Image width

Optionally set the maximum allowed width in pixes for uploaded images. Images over this setting will be automatically resized.

Image height

Optionally set the maximum allowed height in pixes for uploaded images. Images over this setting will be automatically resized

Display

Show or hide the category

Order

In the list component, you can sort by any column. Leave the value by default for the original sort order, or set a custom order with a numeric value.

Category store type

Choose either the file or the database storage scheme for your image files.

This parameter cannot be changed later!

Keep in mind that if you select the database as the storage destination, image files will be stored as BLOB type data which may not be suitable for backup purposes.
