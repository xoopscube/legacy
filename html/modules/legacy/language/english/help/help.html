Getting Started

This section provides a quick start guide with a brief summary of the control panel, dashboard and post-installation checklist.

Quick Start Guide

Once the installation wizard completed, your website is preconfigured with default configuration settings. These settings refer to the pre-existing value of a user-configurable setting that is assigned to your web application platform. Therefore, you need to perform some post-installation tasks before launching your new website. These include setting up the site name and slogan, customizing meta tags (SEO), allowing visitors to register, and user group permissions. More detailed documents are available and will be highlighted throughout this guide.

Store and manage preferences and settings

Save your preferences for all dashboard blocks. Manage your configuration and modules effectively and reliably. Get and deploy add-ons from the X-Update Manager in real time without impacting users, avoiding time-consuming redeployment. The Web App Configuration is built for simplicity, scalability, and security. The web application configuration is built on simplicity, security and scalability.

Control Panel and Custom Dashboard

The control panel, also known as the admin area, gives you access to all actions that can be performed on your web application platform. It also includes a dashboard that can be customized from the "admin panel". You can enable or disable admin blocks, module tips and action tooltips. Your preferences are stored in your browser's localStorage.

Main content

The main idea of having customizable dashboard in XCL is to provide easy access to different sections with a simple click. To achieve this, we picked out the key features of XCL that administrators use most often on a daily basis. We have pre-filled the dashboard with default "admin blocks" that will give you some insights into the environment.

You can add your own Widgets to the Dashboard for different purposes - for example, you can embed any third-party API or Monitoring widgets. In short, Widgets are a quick, non-technical solution, whereas the API is a more involved process that requires specific technical knowledge.

Blocks management

The block management system is built into the kernel and provides custom title, five default block positions, display order, modules, user groups access and cache settings.

Block Title (name)

You can rename the block title, and display or not in your theme.

Block Order (weight)

The display order set in the specific field, next to the block name, will determine its placement on the page, relative to the display orders of all other visible blocks.

The display order is 0, the starting number of 1 and incremented by 1 relates to all subsequent blocks. The new starting number can be changed manually by the user and will then be incremented by 1 against that initial number.

Block Type (custom)

Custom blocks can be created with BBCode and PHP, or HTML, CSS and JavaScript.

Block Side (position)

The block position is just a legacy code convention to identify areas in the theme layout.

Themes can provide several placeholders, for example, by using a prefix added to the block title, or the block's order (weight) value.

Cache setting

You can specify the cache lifetime or prevent a block from being cached, if its content requires updates in real time (e.g. comments or RSS feeds).

Target modules

Where to show the block.

Target groups

user Groups that can access the block.

Modules have several pre-built blocks that offer specific functionality and favor composition over inheritance. These blocks are made available in the block management by enabling modules.

It can also embed code or integrate with third-party features specific to a site's unique needs.

Comment Management

The Comment management system is built into the kernel. It provides a textarea comment box where users can generate their own comments and leave them on a specific piece of content.

You can follow the last comments of your articles on your Dashboard. In addition to having a quick view, you can also give Reply to the comment as well as moderate its status as a public, pending, spam, and delete it.
D3 modules comments are easily integrated (module preferences) and managed through D3Forum.

Category management

Here you can create and edit categories, set the maximum file size, maximum image width and height, display and hide categories and images.

The following options are available when creating a category:

Name

A descriptive name of the category.

Permissions

  • Groups who can upload to the category.
  • Groups who can view the category.

File size

You can optionally set a maximum allowed size in kilobytes for uploaded file. Leave blank if you do not wish to impose a limit.

Example: 1024000 bytes = 1,000Kb = 1MB

Note: Hosting services usually have limits on the amount of data that can be uploaded via PHP If you need to upload very large files, you should contact your server's support service and tell them about the size limits they have put in place. Alternatively, you can install the module Xelfinder, the web file manager can connect to a cloud service.

Image width

Optionally set the maximum allowed width in pixes for uploaded images. Images over this setting will be automatically resized.

Image height

Optionally set the maximum allowed height in pixes for uploaded images. Images over this setting will be automatically resized

Display

Show or hide the category

Order

In the list component, you can sort by any column. Leave the value by default for the original sort order, or set a custom order with a numeric value.

Category store type

Choose either the file or the database storage scheme for your image files.

This parameter cannot be changed later!

Keep in mind that if you select the database as the storage destination, image files will be stored as BLOB type data which may not be suitable for backup purposes.

Word Censor

Control Panel > Preferences > Word Censorship

This section of the Control Panel allows you to enable word censoring. It is also where you define the list of censored words.

Enable censorship?

You may enable or disable word censoring. If you select ‚ÄúYes‚Äù, the system will replace any specified words in channel entries, comments, forum posts, etc. according to your preference below.

Replacement characters

You may optionally specify a word or phrase to be used when replacing censored words. For example, if you set ‚Äútisk tisk‚Äù as your replacement word, and ‚Äúshucks‚Äù is in your censored list, then anytime ‚Äúshucks‚Äù is used it will be replaced with ‚Äútisk tisk‚Äù. If you do not set this preference, a pound symbol will be used for each character that is censored, so ‚Äúshucks‚Äù would be converted to ‚Äú######‚Äú.

Words to censor

Specify a list of words to censor. Wildcards are allowed. For example, test* would censor the words ‚Äútest‚Äù, ‚Äútesting‚Äù, and ‚Äútester‚Äù, while *gress would censor the words ‚Äúprogress‚Äù and ‚Äúcongress‚Äù.

The profanity filter, sometimes referred to as just "filter" or "censor", is a feature typically used on forums and comments that automatically scans users' posts as they are submitted and automatically changes or censors particular words.

Enter the censored words in the textarea using a common delimiter character "|" (pipe) to separate the values.

Set the characters replacing all the censored words in the input field "censor-replace".
