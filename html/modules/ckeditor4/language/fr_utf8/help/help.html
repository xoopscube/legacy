Overview

CKEditor 4 is a rich text editor that allows you to create content directly in your web browser.

This WYSIWYG editor, which stands for What You See Is What You Get, allows you to edit content in a form like the published web page. The module also provides a minimal and secure pre-configured BBCode-based editor to allow users to comment on your content, participate in forums and send private messages. Combined with X-elfinder file manager integration, CKEditor 4 is a major asset to create content and easily manage files.

Main Features

The CKEditor4 module is released by default with a full custom package and provides the following out of the box :

  • Control Panel preferences settings e.g. toolbars, user group permissions, ui color.
  • Localization languages installed by default : English, French, Japanese, Portuguese.
  • Template a Single File Component for frontend and backend.
  • Automatic change of editor HTML or BBCode depending on modules and user group permissions.
  • Automatic switch of ToolBar based on modules preferences and user group permissions.
  • BBCode editor extends the CKEditor standard bbcode plugin
  • HTML editor with custom Toolbar for each user group
  • Extra Plugins customized CodeMirror, oEmbed, Paste (raw text, formatted or code).
  • PHP mode for PHP code blocks without the <?php opening tag.
  • Smarty mode for Smarty Template Engine code blocks.
  • elFinder open-source web file manager with cloud storage settings.

CKEditor4 module Preferences

  • Toolbar for Admin : CKEditor's "config.toolbar" for admin.
  • Toolbar for registered user : CKEditor's "config.toolbar" for registered user.
  • Toolbar for guest : CKEditor's "config.toolbar" for guest.
  • Toolbar for BBCode plugin : CKEditor's "config.toolbar" for BBcode plugin.
  • Toolbar UI Color : Setting Editor UI Color with hexadecimal color code eg. #405060
  • CSS URL of editing area : URL of external CSS applied to edit area is set up per line.
    * A special value to automatically parse the CSS specified in the HTML <head> of a page.
  • Extra plugins : CKEditor's "config.extraPlugins" setting.
  • Custom config URL : CKEditor's "config.customConfig" setting.
  • Allow All Content : CKEditor's "config.allowedContent" setting.
  • Auto Paragraph : CKEditor's "config.autoParagraph" setting.
  • Enter Mode : CKEditor's "config.enterMode" setting.
  • Shift+Enter Mode : CKEditor's "config.shiftEnterMode" setting.
  • X-elFinder : Set X-elFinder file manager directory name for server browser.

Implement CKEditor to textarea

  1. Clone/Duplicate your Templates to create a new Set.
  2. Edit any installed module's templates with a textarea and add a new parameter editor=html or editor=bbcode
Editor HTML
<{xoops_dhtmltarea editor=html}>
Editor BBCode
<{xoops_dhtmltarea editor=bbcode}>
Editor PHP

The example bellow implements the WYSIWYG editor for custom blocks type PHP by using raw data and escape method to make the form more secure. See the implementation in admin templates e.g. blocks_admin_edit.


<{if $block.is_custom}>
    <{if $block.ctype == 'P'}>

        <!-- Textarea Block raw data and escape method -->
        <textarea name="contents[<{$block.bid}>]" id="textarea_content" rows="20">
        <{$block.content_raw|escape}>
        </textarea>

    <!-- Configuration for PHP -->
    <script type="text/javascript">
    CKEDITOR.replace( 'textarea_content', {
        customConfig: '<{$xoops_url}>/modules/ckeditor4/ckeditor/config_php.js',
    });
    </script>

    <{/if}>
<{/if}>
Note ! Remember to update your new template set.
The CKEditor4 module won't make any data conversion.
Any textarea with previous data will be automatically escaped.

Customize CKEditor4

CKEditor 4 comes with a rich set of configuration options that make it possible to customize its appearance, features, and behavior.

CKEditor4 Toolbar

There are several approaches to CKEditor 4 toolbar configuration to choose from:

  • Using the toolbar configurator — the most recommended and easy to use solution.
  • Toolbar groups configuration — requires manual toolbar configuration (for advanced users only)
  • “Item by item” configuration — requires manual toolbar configuration (for advanced users only)

Most important settings can be defined in CKeditor preferences, including Toolbar customization for each group.

  1. Select a special group (checkbox)
  2. Example of toolbar for special group :

User Interface color

The CKEditor UI color can be adjusted from the module's preferences Admin > CKEditor > Preferences to match the look and feel of a website or an application. In order to define the color of the user interface, use the config.uiColor configuration setting which accepts an RGB color code.

The editor instance UI color was set by default to the theme RGB value.

Please note that this configuration option can only be used for editor skins that are compatible with the so-called "chameleon" feature, for example:

  • Moono-Lisa
  • Moono
  • Kama
Note :
Refer to the Setting Editor User Interface Color article to learn more about this feature : uicolor ⭧

CKEditor Plugin : Code Snippets

This plugin lets you insert rich code snippets with syntax highlighting into the editor. The default implementation uses the integrated highlight.js library, but the plugin is easily customized to use a different syntax highlighter e.g. XCL common PrismJS.

Additionally, the you can also use the following configuration options:

  • config.codeSnippet_theme - to change the default highlighter theme.
  • config.codeSnippet_languages - to change the list of supported languages.

Syntax highlighting of over 50 programming languages (note that SVG and XML are under the HTML category, and JSON is under JavaScript)

PrismJS

Prism is a lightweight, extensible syntax highlighter, built with modern web standards in mind.

PrismJS 1.30.0 is installed in /common/prismjs/ and preload by the theme.html

Visit online builder to customize Default Minified version ⭧
