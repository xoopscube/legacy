X-Update Manager

X-update Manager is a software update service (SUS) with a user-friendly GUI to quickly extend and update your web app. This module can download all critical updates and security patches to your server as soon as they become available. In addition, you can manage your FTP/SSH credentials in the module preferences.

With the introduction of X-update, site administrators just need to log in to the administration Control Panel to install or update modules, themes and preloads. The deployment is as fast as possible by downloading archives from repositories such as GitHub or GitLab. X-update also provides support for language catalogs (localization) and multiple instances of D3 modules (duplicatable and directory free).

X-Update automatically monitors the public repository for updates as well as installed modules and themes.

Available updates are highlighted in the top notification, and the front-end if X-Update's block is installed.

If your SSL certificate is not valid, check cURL: Disable SSL verification in module's preferences.

Module

Modules can add new functionality to your Web Application Platform or enhance and extend existing functionality.

There are different types of modules (utility, service, etc.) usually classified as follows :

Category Description
Core Modules required by the Bundle Package
Trust module D3 modules running multiple instances
X2 Modules Single instance

Core modules

The default settings ensures that required core modules are running and redirects to the built-in installation wizard if any required module is missing in your application.

Trust modules

D3 nodules are installed in the Trust_PATH directory. These modules are duplicable and allow the installation of several instances with different names, language and templates on the client side.

X2 Modules

These modules are installed in the public side and run a single instance. The core modules, legacy System and Renderer, are designed to ensure backward compatibility and easier upgrades from previous versions. Thus providing forward compatibility with the extensible built-in features such as delegate and preload (hooks, extensions).

Theme

This section provides a brief introduction to download and install Themes to customize the global appearance of your website.

The core Render is framework-agnostic, which means Themes can be built with different CSS and JavaScript frameworks.

  1. Select a Theme and follow the wizard steps
  2. Browse to ¬ª Theme Management
  3. Select your Default Theme

Theme Management allows you to "Select" the default theme and to select multiple themes available to users.

Allow Users to select Theme

Any theme that is "checked" in Theme Management will be available to users in the block "Themes".

If you install the block "Themes" and grant users permission, they can then select their favorite theme and customize the look and feel of the website.

Preload

This section provides a brief introduction to Preloads and walks through the installation of a simple extension.

Preloads are commonly Single File Components that extend the Web Application Platform with customized functionality:

  • Productivity workflow
  • Web page content enrichment
  • Information aggregation
  • Custom UI components

In a nutshell, Preloads are simple extensions, or plugins, to add new functionality, enhance the current usability or customize a design without hacking the core system.

Enable and disable a Preload

Unlike modules, Preloads do not have a control panel.

A Preload is installed in the public directory /preloads/

To disable a preload, the best practice is to use X-elFinder
The file manager can be used to delete or move the preload file to a directory /preloads/disabled/

Preferences

X-Update manager provides several options to get and deploy add-ons :

  • FTP, Login password for FTP or passphrase of private key for SSH2 Key-Auth
  • Private key for SSH connection
  • Write permissions of files and folders
  • Update only used Language
  • Disabled items from Store list
  • Select a Tag module
  • Preload files edited with X-elFinder
  • View the debug output
  • Add-on Store URL
  • HTTP maximum connections
Disable SSL verification : Turn off cURL's verification to bypass SSL certificate validation for local and test servers.
