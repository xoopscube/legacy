概要

ユーザーモジュールは、シンプルで安全で拡張性の高いアカウント認証システムを確保するためのベストプラクティスを提供します。

このモジュールを使用すると、ユーザー、グループを管理し、デフォルトのセキュリティレルムで定義された権限を設定することができます。

次の機能が用意されています:

  • 認証
  • ユーザー登録
  • ユーザープロファイル編集
  • ユーザープロファイルの表示
  • ユーザーについてのブロック
  • ユーザー管理
  • その他のオプション

ユーザーは、認証できるエンティティです。 各ユーザーには、レルム内で一意のidが割り当てられます.

多数のユーザーの管理を容易にするために、ユーザーを名前付きグループに編成することができます。

ユーザーには、他のグループのメンバーシップを割り当てることができます。

管理者権限を持つのは、Webmasterグループのメンバーとしてログインしているユーザーのみです。

ユーザーは、さまざまな基準に従って他のユーザーを検索できます

ユーザーは、プライベートメッセージモジュールを介して他のユーザーや電子メールにプライベートメッセージを送信することができます。

グループ管理

ユーザーを複数のグループに割り当て、モジュールとブロックのコンテンツへのアクセス権を付与できます。

インストールウィザードでは、次の必要なグループが作成されますs :

  • ウェブマスター（管理者）
  • 登録ユーザー（メンバー）
  • 匿名ユーザー(ゲスト)
必要なグループを削除することはできません！
新しいグループを作成し、作成された新しいグループごとに権限を変更できます。

The group membership remains independently of the session (the logging session). 継承または除外の概念は、グループメンバーシップを管理するのに十分な権限を持つユーザーが手動で処理する必要があります。

役割

Idベースのグループは加法的であり、これはidがユーザーが属するすべてのグループの合計（または結合）であることを意味します。

ロールは、特定のユーザーに権限を追加するために使用されます。 :

  • Webmaster-ユーザーと権限を割り当てることができます
  • モデレータ-フォーラムのみを管理できます
D3モジュールまたはTrustモジュールには、個々のユーザーに割り当てることができる機能があります。

D3フォーラムの権限セクション、およびその他の信頼モジュールでは、対応するチェックボックスを選択できます 特定のユーザー、つまり表示、投稿、編集などに付与できる権限には、次のようなものがあります。

セキュリティとアクセスのニーズを考慮し、それらのニーズに応じてグループを設定します, 次に、付与する権限に基づいてユーザーをグループに追加します。

ユーザー管理

Webサイトへのアクセスは、ユーザーがグループに割り当てられるグループシステムによって制御されます。 グループ権限は、モジュールとブロックのコンテンツへのアクセスを許可します。

D3モジュールまたはTrustモジュールは、ロールベースのシステムを提供して個々のユーザーに権限を割り当てることで、ユーザー管理を拡張できます。

ユーザーの権限は、ユーザーが属するすべてのグループ権限の合計によって決定されます。

ユーザーアカウント

ユーザーアカウントには、既定でログイン情報、認証、およびプロファイルフィールドがあります。 ログイン資格情報(通常はユーザー名とパスワード)は、アカウントにログインしているユーザーを認証します。

ユーザーアカウントは、ユーザー登録とアクティベーションによって追加されます。 ウェブマスターはuser managementモジュールでアカウントを作成できますが, また、新しいフィールドを追加することで、デフォルトのユーザープロファイルを拡張します。

アバター

登録ユーザーは、必要に応じてアバターと呼ばれる彼らのオンラインペルソナをファッションすることができます。

このオプションは、モジュールユーザーグループの"環境設定"からオフに設定できます。

アバターの種類。

一つは、管理者が一連の画像をアップロードすることによって準備するシステムアバターです。

もう一つは、ユーザーが自分の好きな画像ファイルをアップロードするためのカスタムアバターです。

セキュリティ上の理由から、許可される唯一のファイル形式は次のとおりです :
.GIF, .JPG and .PNG

ログイン
概要

このブロックは、まだ認証を通過していないゲストユーザーに対してのみ、ログインフォームを表示します。また、新規ユーザー登録が設定で許可されているならば、それは register.php へのリンクを表示します。さらに、SSLログインが許されているなら、一般設定で指定されたリンクを表示します。

備考

もしあなたが間違ってこのブロックをアンインストールしてしまっても、 user.php へアクセスすれば、ログインブロックなしでログインを行うことができます。

オンライン情報
概要

ユーザーモジュール - 一般設定

新規ユーザの登録を許可する

ゲストがあなたのサイトに登録を行うことができるかどうかをしています。「はい」を選択した場合、 register.php が利用可能になり、ログインブロックが register.php へのリンクを表示し始めます。たとえ「いいえ」が選択されていた場合でも、あなたは新しいユーザーをユーザー管理から追加することができます。

パスワードの最低文字数

パスワードの最低文字数を指定します。長いパスワードはSpecifies the minimum length of password. 長いパスワードは、パスワードを推測するのをより困難にします。 もちろん、長さは強いパスワードのすべてというわけではありませんが、５文字以上を指定するべきでしょう。

ユーザ名の最低文字数と最大文字数

ユーザー名に最低必要な文字長を指定します。このオプションはセキュリティに関するものではありません。 単純に好みで決めてください。

ユーザ自身のEmailアドレス変更を許可する

ユーザがユーザアカウントの編集(edituser.php)で、彼らのEメールアドレスを変えることができるかどうかを指定します。実のところ、このモジュールはEメールアドレスを変更するユーザーに対して、確認のメールを送りません。

新規ユーザ登録の際にメールにて知らせを受け取る

このオプションが「はい」である場合、次のオプションで設定されたグループは新しいユーザー登録に関して通知を受け取ることができます。

通知先グループ

先のオプションが「はい」と設定されていた場合、ここで指定したグループは通知メールを受け取ります。このオプションは、通常、管理者グループを指定するとよいでしょう。

新規登録ユーザアカウントの有効化の方法

登録後に、ユーザーはアクティベーション（有効化）されなくてはいけません。アクティベーションは「ログイン可能にする」ということを意味しています。ユーザーアクティベーションを行う方法はいくつかあります。

ユーザー自身の承認が必要

これはウェブサイトシステムにおいて疑いなく世界で一番ポピュラーな設定です。ユーザーが登録を行う際、彼らはアクティベーションメールを得ます。そして、彼らはそのメールに書かれているアクティベーション URL をクリックすることによってログインが可能な状態になります。ほとんどの場合、この設定を選択しておけば間違いがないでしょう。

自動的にアカウントを有効にする

簡単に言えば、アクティベーション機構が全くない設定で、すべてのユーザーアカウントが登録と同時に有効になります。しかし、これを使用する場合は、あなたは BOT に注意するべきです。 BOT は自動的に登録を行い、 SPAM を送りつけてくる可能性があります。したがって、この設定はイントラネットなどの閉じたセグメントで有効でしょう。

管理者が確認してアカウントを有効にする

次のオプションで指定されたグループから直接承認を受けることによってのみ、ユーザーはアクティベートされます。要するに手作業ですから、この選択は承認者にきつい仕事を強制するかもしれません。

ユーザ名として使用可能な文字の設定

ユーザー名に関してどの程度の自由度があるのかを以下の設定から選択します。

強（アルファベットおよび数字のみ）

この選択では、あなたのサイトは以下の文字列のみをユーザー名に使用できる文字として許可します。

  • 大小アルファベット
  • 数字
  • アンダースコア (_)
  • ハイフン (-)

中

この選択では、あなたのサイトは以下の文字列のみをユーザー名に使用できる文字として許可します。

  • 大小アルファベット
  • 数字
  • アンダースコア (_)
  • ハイフン (-)
  • 小なり(<)
  • 大なり(>)
  • カンマ(,).
  • ピリオド(.)
  • ドル記号($)
  • パーセント(%)
  • シャープ(#)
  • アットマーク(@)
  • エクスクラメーションマーク(!)
  • シングルクォーテーション(')
  • ダブルクォーテーション(")

低 - ２バイト文字を許可する

この選択では、あなたのサイトはコントロールコード以外のすべての文字を許可します。 しかし、この選択はいくつかのモジュールで問題を起こすかもしれません。 したがって、ユーザーモジュールはこの選択を推奨していません。

アバター画像のアップロードを許可する

ユーザが彼のアバターとして彼のお気に入りの画像ファイルをこのサイトにアップロードできるかどうかを指定します。たとえ「いいえ」が選択された場合でも、ユーザーは管理者が事前に準備するシステムアバターから彼のアバターを選択することができます。

アバターアップロード権を得るための発言数

もしカスタムアバターのアップロードを許可しないのであれば、あなたはこのオプションに触れる必要はありません。

これはカスタムアバターをアップロードする権利をユーザーが獲得するために必要な投稿数の設定です。これはナンセンスかもしれません。 0 を設定するのがよいでしょう。

アバター画像の最大幅(ピクセル)

もしカスタムアバターのアップロードを許可しないのであれば、あなたはこのオプションに触れる必要はありません。

カスタムアバターに幅の制限を設定します。ユーザがこの値より横幅の広い画像をアップロードしようとする場合、このモジュールはエラーメッセージを表示します。

アバター画像の最大高さ(ピクセル)

もしカスタムアバターのアップロードを許可しないのであれば、あなたはこのオプションに触れる必要はありません。

カスタムアバターに高さ（縦幅）の制限を設定します。ユーザがこの値より縦幅の広い画像をアップロードしようとする場合、このモジュールはエラーメッセージを表示します。

アバター画像の最大ファイルサイズ(バイト)

もしカスタムアバターのアップロードを許可しないのであれば、あなたはこのオプションに触れる必要はありません。

カスタムアバターにファイルサイズの上限を設定します。ユーザーがこの値より大きいファイルサイズの画像ファイルをアップロードしようとする場合、このモジュールはエラーメッセージを表示します。

ユーザが自分自身のアカウントを削除できる

ユーザがユーザアカウントの編集において、自分のアカウントを削除することができるかどうかを設定します。はい、ユーザーはユーザーアカウント情報(userinfo.php)からのリンクを通じて、アカウント削除のダイアログへ進むことができます。削除に関するこのダイアログが非常に簡素なものになっているので、あなたはそれらのテンプレート user_delete.html と user_delete_success.htmlを編集するべきでしょう。

アカウント削除前の確認メッセージ

これはアカウント削除ダイアログで表示されるメッセージです。もし先のオプションで、ユーザが自分自身のアカウントを削除できる設定を行っていないのであれば、あなたはこのオプションに触れる必要はありません。

ユーザ名として使用できない文字列

これはあなたのサイトがユーザ名として使用することを許さないパターンです。 | で区切ってください。これは大文字と小文字を区別せず、正規表現による指定が可能です。加えて、このルールはコントロールパネルのユーザー管理画面では適用されません。

もしゲストユーザーの新規ユーザー登録を許可していないのであれば、あなたはこのオプションに触れる必要はありません。

ユーザのemailアドレスとして使用できない文字列

これはあなたのサイトが電子メールアドレスとして使用することを許さないパターンで、フリーメールによるユーザー登録を防止するような場合に使用します。 | で区切ってください。これは大文字と小文字を区別せず、正規表現による指定が可能です。加えて、このルールはコントロールパネルのユーザー管理画面では適用されません。

利用許諾文を表示する

ユーザー新規登録時に、次のオプションで指定される利用許諾文を表示するかどうかを指定します。

利用許諾文

これはユーザー新規登録において、ゲストに対して表示される注意書きです。もし前のオプションで「いいえ」を選択しているのであれば、あなたはこのオプションに触れる必要はありません。

ユーザ名の保存に使用するクッキーの名称

ウェブブラウザに対するクッキー名を指定します。高度な設定であり、ほとんどの場合、あなたはこれを変更する必要はないでしょう。

ログインにSSLを使用する

あなたは、SSLログインを有効にするために、特別な追加ログインスクリプトをインストールしなければなりません。「はい」を選択した場合、ログインブロックは次のオプションで指定されたリンクを「ＳＳＬログイン」として表示します。これらのオプションは XOOPS2 JP のものと互換です。おそらく、あなたは SSL ログインに関して、これよりベターな解決策を見つけることが可能です。追加スクリプトは /extras ディレクトリに入っています。

SSLログイン時に使用するPOST変数の名称

このオプションも SSL ログインのためのものです。 POST リクエスト変数の名前を指定します。

SSLログインページへのURL

このオプションも SSL ログインのためのものです。あなたがインストールした追加ログインスクリプトの URL を指定します。
