ユーザーモジュール - 基本知識

初めに、あなたはユーザー、グループ、ユーザーランク、アバター、パーミッションという５つの基本的な要素を理解しなければなりません。このセクションはそれらの基本をあなたに説明します。より詳細はそれぞれのセクションを読んでください。

ユーザー

これはユーザアカウントを意味しています。 ユーザアカウントには、 XOOPS 認証のためのログイン情報と、固定のプロフィールフィールドがあります。ユーザー名、電子メールアドレスとパスワードは、サイトを動かすための必須項目になります。ユーザーアカウントは管理機能かユーザー登録とよって追加されます。

グループ

これはユーザアカウントのグループを意味しています。ユーザアカウントは複数のグループに所属することができます。 グループには、モジュールのためのパーミッションがあります。 このモジュールで、パーミッションを持っているものはユーザではなく、グループです。ユーザのパーミッションはユーザが所属するグループによって決定されます。

あなたは、"パーミッション管理" と "ユーザーをリストすること" の両方のためにグループを創設できます。 しかし、あなたは３つのシステム固定グループについて知っておかなければなりません。これらの特別なグループは XOOPS Cube Legacy のインストール時に作成されており、削除することはできません。

サイト管理者

このグループに所属するユーザーは何でもすることができる特別なパーミッションを持っています。言い換えれば、このグループに属することが可能なユーザーはウェブマスターだけということです。このグループがすべてのパーミッションを持っている特権グループであることに注目してください。たとえばあなたが自分の友達にサイト管理の "一部分" を頼む場合には、その友人をこのグループに加えるべきではありません。

もう一度言います。このグループはすべてのパーミッションを持つ、何でもできる特権グループです。

Registered Users

通常の一般的なユーザはこのグループに属します。 このモジュールが提供するユーザー登録機能は、新しく作成されたユーザーが所属する最初のグループとしてこのグループを使用します。この性質上、あなたは管理パーミッションをこのグループに与えるべきではありません。

ゲスト

ログインを行っていないアクセス（訪問者）はこのグループとして扱われます。このグループは本当に特別な、そして仮想的なグループの一種です。このグループはメンバーを持つべきではありません。

ランク

ランクはユーザプロフィール情報のうちの１つです。ランクはシンボル画像とランク名を持っています。 ユーザのランクは彼の投稿数よって決められます。 このランクという機能は、あなたのサイトをおもしろく演出するための一種のジョークです。ですから、あなたはランクが XOOPS の認証やパーミッションに何らかの影響を与えるのではないかという心配をする必要はありません。

アバター

アバターもまた、ユーザープロフィール情報の１つです。これは個人のグラフィカルなラベルです。このモジュールは２種類のアバター機能を提供します。ひとつは管理者があらかじめ数種類の画像を提供するシステムアバターです。もうひとつはユーザーが好みの画像ファイルをアップロードできるカスタムアバターです。

パーミッション

パーミッションは、グループの行動許可（≒権限）です。このモジュールは基本的なパーミッションを編集できます。もしあなたがモジュール固有のパーミッションを編集することを除くのであれば、それぞれのモジュールが持っているパーミッション編集機能を使用してください。

最終的なユーザのパーミッションは、そのユーザが属する全グループのパーミッションの総和（すべてを足したもの）で決定します。
