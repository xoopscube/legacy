Overview

User Groups enables you to manage privileges and assign permissions to user groups defined within the default security system.

The User module provides recommended practices for establishing a simple, secure, and scalable system for authenticating user accounts.

The following features are provided by the modules Users, Message and Protector :

  • Group-based permissions
  • Registration : new user account and activation method
  • Authentication and security limit of Log in attempts
  • Extend User Profile with custom fields
  • Private Message
  • Blocks : Login, Users Online, New Members, Top Posters
  • Manage subscription event notification

Group Management

Groups are used to segment users within your web application. You can assign users to multiple groups and grant them access to modules and block content. The installation wizard automatically creates the following default User Groups and Roles:

User GroupsDescription
WebmastersFull administrative permissions across the application
Registered UsersAll users except anonymous. Permissions to read and write comments.
Anonymous UsersGuest users or visitors, by default anonymous. Permissions restricted to read.
RolesPrivilege
WebmasterAdministration
ModeratorForum moderation
These User Groups and Roles are required by the system and cannot be deleted !
You can create new Groups and modify the permissions for each new Group created.

Actions

Assign a user to the Group.

Edit the group's name and write a short description.

View properties of the group's global permissions.

Set group permissions for modules and blocks.

Creating Groups

Browse to Group Management and click the button :
Add new group

  1. Enter the name of the user group in the Name field.
  2. Write a short description
  3. Click Submit

The Group Management page is displayed with the new group.

Click Group Permissions to modify the Group.

Alternatively, click Delete to delete the user group.

It's recommended to start with the same default permissions of the group Registered Users.

Assigning Permissions to User Groups

The ability to perform an action requires privileges. While permissions and privileges are often used interchangeably, they technically have distinct meanings in web application security. Modules expose permissions, and users possess privileges. Privileges are essentially assigned permissions granted to User Groups. Therefore, when you assign a permission to a user group, you are granting a privilege to all its members.

Declaration of the actions that can be executed on the module.

User Account

Access to your website is controlled by a group system by which users are assigned to User groups. Group permissions allow access to the content of modules and blocks.

User accounts are added through user registration and activation. Although Webmasters can create and activate accounts in User Management.

D3 modules, or Trust modules, can extend User Management by providing a role-based system to assign individual permissions.

A user's permissions are determined by the sum of all group permissions to which the user belongs.

User Profile

A user account has login information, authentication, personal information fields, private messages and notification management.

Registration

Registration to the site is allowed by default. If you are running a closed site, or need to temporarily disable registration, you may disable this feature by switching "Allow new user registration" to the "No" setting in the module preferences :
Control Panel »» User Groups »» Preferences

Activation

Activation is a process of verifying a registered user. The registered user's account can be automatically activated upon registration. Indeed, an activated account is required to connect. However, other activation modes are available.

The Activation mode specifies the activation workflow :

  • Activate automatically : This mode will automatically activate a user upon first registration.
  • Require activation by user : Activation by following the link in the confirmation message sent to the user's email address.
  • Activation by administrators: Activation by groups selected in the module preferences or by administrators via the back-end area.

Users have the ability to resend the activation email by clicking Send the verification email.

Avatar

This feature allows Users to fashion their online personae called avatars. See section Avatar Management.

Log in

Login credentials, typically a username and password, authenticate a user logging into an account.

Users can also reset a lost or forgotten password.

Limit Log in Attempts

This security feature blocks an Internet address from making further attempts after a specified limit on retries is reached, making a brute-force attack difficult or impossible. If a user experiences too many failed sign in attempts, their account will be temporarily suspended for a period of time. The module Protector allows administrators to enable IP blocking on their web application, in order to block all known malicious IP addresses to reduce the risk of an attack.

Private Message

This feature is implemented by the module Message.

Notifications

This feature is implemented by the core system.

Extend the User Profile

The User Profile can be easily extended by adding new fields with the module Profile. You can add custom fields for your users, set permissions and requirements. This feature can be used if you need to add additional fields that are not available by default.

Avatars

Registered Users can optionally fashion their online personae called avatars.

Enable this option in the module's preferences. "Allow new user registration" to the "No" setting in the module preferences :
Control Panel »» User Groups »» Preferences

Types of avatars

Allow users to select an avatar from a previously uploaded collection.

Or allow users to upload their own avatar :

  • Avatar image max width (pixel)
  • Avatar image max height (pixel)
  • Avatar image max filesize 1 Megabytes (MB) = 1,048,576 Bytes

For security reasons, the default files extensions allowed for avatars are : .GIF, .JPG and .PNG

Some Themes can generate automatically an avatar based on the first letter of the username.

Module Blocks

Login

This module block shows the login-form for only anonymous users who still are not passing the authentication. If the user registration is allowed, it shows the link to create an account. The SSL login option ensures backwards compatibility.

Create An Account is what the anonymous user do after the sign-up. An account needs to be associated with a signature or profile, and often that can't be done until the user completes the registration process. That registration process may require them to agree to the “Terms and Conditions” : that's the sign-up part.

Note : You can log in using example.com/user.php, if you have uninstalled this block by mistake.
Users Online

This block displays the names of users likely to access your site and how many are online.

Getting correct information for online users is too cumbersome and resource consuming process. Therefore, this block does not attempt to display an exact result. But this should give you some insight.

New Members

This block displays the names of users who have recently registered on your site.

Options
Display x members
Specifies the limit of recently users to display.
Display member avatars
If YES, displays user avatars with users name.
Top posters

This block shows the ranking of the top contributors.

Options

Display x members
Specifies the limit of users to display.
Display member avatars
If YES, displays user avatars with users name.
Do not display users whose rank is x
You can remove users belonging to specific groups from the ranking.

Ranks

The Ranking system displays user rank next to each user’s name based on the number of posts.

By default there are two user special ranks :

  • Webmaster
  • Moderator

You can create new user ranks with new rules and images. There is no limit to the number of ranks that you can create.

Once you have added ranks, visit a profile or page with comments to see it in action.

Mail Job

The mail job allows you to create job definitions that send simple email, or notifications. You can use MailJob to inform users of events, for example :

  • User conferences.
  • Networking events.
  • Learning Workshops.
  • Community events.
  • Outdoors activities.
  • Auctions and Pop-up shops.
  • Special guest events.

Usage

Add an empty job from the control panel.

Specify a name and a label for the job.

In a next step invoke the job wizard to find users. The wizard brings up the following window :

Search Users filter according to various criteria.

Then, you can manage the job messages to users :

Retry, resubmit, and expiration.

You can customize the maximum number of retries.

This gives you the option to continue processing new events.

Preferences

Features in Control Panel

This article provides information specific to user management.

User module - Preferences

Allow new user registration?

Specifies whether anonymous users can register to your site. If YES, registration is available, and Login Block shows the link to create an account (register.php).

Even if registration is deactivated, you can still add a new user in the user management.

Minimum length of password required

Specifies the minimum password length. A long password makes a password harder to guess. Of course, length is only one strong password factor. But, you should specify at least five.

Minimum length of username required

Specifies the minimum username length. This option is not about security. This is a decision to be made based on the needs of your organization.

Allow users to change email address?

Specifies whether users can change their email address in user profile editor (edituser.php). By the way, this module does not send a confirmation email for users who change their email address.

Notify by mail when a new user is registered?

If this option is Yes, the group specified by the following option can receive an e-mail notification about new user registrations.

Select a group to which new user notification mail will be sent

Specifies the group that receives the notification mail in case the previous option is Yes. The webmaster group is usually selected for this option.

Select activation type of newly registered users

After registering, a user must be activated. Being activated means being able to log in. There are several ways to activate users.

Require activation by user

This is far and away the most popular setting in the world of website systems. A user gets an activation email when they register. They are then able to login by clicking on the activation URL on the mail. This choice is satisfactory in most cases.

Activate automatically

In simple terms, there is really no activation mechanism, everyone is activated straight away. But, if you use it, you have to be careful of internet bots. An Internet bot, web crawler, bot, or simply bot, is a software application that performs automated tasks over the Internet. A bot can auto-register and SPAM. Therefore, this choice is only recommended for a closed segment.

Activation by administrators

A user can only be activated by direct approval of the group specified by the following option. This activation requires human intervention, and therefore seems more effective. However, this can be a time-consuming task for activators.

Select the level of strictness for username filtering

Select the strictness level for the username from the following options.

Strict (Only alphabets and numbers)

This option only allows the following characters for username.

  • Alphabets without case-sensitive.
  • Numbers.
  • Under score (_).
  • Hyphen (-).
Medium

This option allows the following characters for username.

  • Alphabets without case-sensitive.
  • Numbers
  • Underscore(_).
  • Hyphen(-).
  • Less than character(<).
  • Greater than character(>).
  • Comma(,).
  • Period(.).
  • Dollar($).
  • Percent(%).
  • Sharp(#).
  • At sign(@).
  • Exclamation point(!).
  • Single quotation(').
  • Double quotation(").
Light - allow 2byte characters

This option allows all characters except control code. But this can be a cause of problem for some modules. Therefore, this option is not recommended if you are using old modules.

Allow custom avatar upload?

Specifies whether a user can upload their favorite image file as an avatar. Even if the case is No, a user can choose an avatar among the avatars made available by the administrators.

Minimum posts required

If you allow uploading of custom avatars, you don't need to set this option.

This is the minimum number of posts required for users to upload a custom avatar. This decision depends on your organization's community member ranking strategy.

Avatar image max width (pixel)

Define if custom avatar upload is allowed.

Specifies the limit width for the custom avatar. In the event that a user uploads an image larger than this value, this module displays an error message.

Avatar image max height (pixel)

Define if custom avatar upload is allowed.

Specifies the limit height for the custom avatar. In the event that a user uploads an image larger than this value, this module displays an error message.

Avatar image max filesize 1 Megabytes (MB) = 1,048,576 Bytes

Define if custom avatar upload is allowed.

Specifies the limit filesize for custom avatar. In the event that a user uploads an image larger than this value, this module displays an error message.

Allow users to delete own account?

Specifies whether users can delete their own account in the user profile edit. If Yes, users can access the account deletion dialog via the user profile link (userinfo.php). Since these delete dialogs are very minimal, you can improve it with a notice by changing the templates --- user_delete.html, user_delete_success.html.

Execution confirmation message

This is the message displayed in the account deletion dialog. If you don't allow users to delete their own account, you don't need to edit this option.

Enter names that should not be selected as username

These are patterns that your site does not allow as usernames. Separate them with a | (pipe). This is case-insensitive and regex enabled. This rule does not apply to user management in the control panel. If you don't allow users to delete their own account, you don't need to edit this option.

Enter emails that should not be used in user profile

These are patterns that your site does not allow as email address. Separate them with a | (pipe). This is case-insensitive and regex enabled. This rule does not apply to user management in the control panel.

Display disclaimer?

Specify whether the user registration displays the disclaimer specified by the following option.

Registration disclaimer

This is the disclaimer stated in the user registration for the anonymous user. If the previous option is No, you do not need to edit this option.

Name for user cookies

Specifies the cookie name for web browsers. In most of the cases, you don't need to change this.

Use SSL for login?

You need to install the additional login script to enable SSL connection. If YES, Login Block shows the link "SSL Login". These options are for compatibility with XOOPS2-JP. You may be able to find a better solution to upgrade and migrate your SSL connection. The script is in the public "/html" directory.

SSL Post variable name

This option is for SSL Login, too. Specifies a name of the POST-request variable.

URL where SSL login page is located

This option is for SSL Login, too. Specifies the URL where you have installed the additional login script for SSL.
