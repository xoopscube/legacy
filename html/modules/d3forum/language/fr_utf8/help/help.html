D3 Forum

D3 Forum is a duplicable module used to create and manage forums, or message boards discussions. This module provides features comparable to popular community forum software. It also provides the comments-integration functionality for D3 modules e.g. Pico.

Creating discussion forum categories helps managers and users to quickly access the required information. Generally, the structure of a forum covers particular categories. See more about this in documentation section structure.

Administrators can assign user groups and individual permissions into various categories. Users with appropriate access rights can discuss one specific topic with other members of the platform.

Usage

D3 modules are designed to let you create categories and content on the front-end, while user permissions, customization and module preferences are managed on the back-end.

First, create a category to organize your forums and help users to quickly join discussions. Next, create a forum Example for structuring forum categories

Create a New Category

Click "New Category" and fill in the details in the form page.

  1. Category title
  2. Category description (allows format text with BBCode)
  3. Parent category (default top-level category)
  4. Category order (weight position, default sorting by category ID)
  5. Category options (more about this below in section "Overridable options").

When you‚Äôve included all these details, you can simply click "Submit" at the bottom of the form, and your Category will be created.

Create A New Forum

Select a category and click "Admin" (or "Action", depending on your theme) and "New Forum"

Fill in the details for the new forum.

  1. Forum title
  2. Forum description (allows format text with BBCode)
  3. Parent category (default top-level category)
  4. Forum order (weight position, default sorting by category ID)
  5. Format for comment-integration. Leave blank for a default forum. Modules with comment-integration will set this field automatically.
  6. Forum options (more about this below in section "Overridable options").
  7. Click "Submit"
Category, subcategories and forums inherit permissions from the nearest parent category, in the absence of individual permissions, from the Top-level category, by default.

Moderation

Only administrators can create categories and forums. However, modules based on D3 architecture design you can assign users and roles to moderate and create forums (either individually, or for all categories on your module instance). For example, you may grant users a specific role "moderator" with the ability to create forums.

  1. Click the button, or icon "Edit category"
  2. Click "Category permissions" to manage users permissions

Overridable options

Categories, subcategories and forums inherit their options from the Top-level category, by default. However, you can customize the default options when creating or editing a category or forum.

Override preferences by writing a new line for each option.
(option name):(option value)

Example to show breadcrumbs (1=on, 0=off):
show_breadcrumbs:1

Overridable options current values

default_options:smiley,xcode,br,number_entity

Default filters for content. Specify filter names separated by comma ","
Available options: smiley,xcode,br,number_entity,special_entity,html attachsig,u2t_marked

allow_html:0

Enable HTML (0 = off, 1 = on). Beware of the risks of Script injection attack by malicious users. Allow trusted user groups only.

allow_textimg:0

Enable external images in comments and posts. These images, sometimes called Web Beacons, can be used to track IPs or User-Agents of registered users. External images are not a security threat. However, it is recommended to turn off external images.

allow_sig:1

Enable Signature. Users can add their profile signature in the comments.

allow_sigimg:0

Enable external images in signature. You can turn off automatic loading of external images as an additional privacy protection.

posts_per_topic:20

Set the maximum posts per topic.

hot_threshold:10

Enable Hot Topic Threshold. Set the number of replies that a topic must receive for it to be considered a "hot topic".

topics_per_page:10

Set the maximum topics per page.

use_vote:1

Enable Voting

use_solved:1

Enable Solved

allow_mark:1

Enable Marking

allow_hideuid:0

Allow Users to Post Anonymously. Registered users can check the anonymous option when they write topics and posts.

viewallbreak:10

Topics per page in the view forum.

selfeditlimit:240

Time limit to edit posts. Disable with value set to 0. Default in seconds (4min): 240

selfdellimit:0

Time limit to delete posts. Set the value in seconds. Disable delete with value set to 0.

css_uri:{mod_url}/index.php?page=main_css

CSS file for this module 'main.css'

images_dir:images

Directory for image files. Relative path to module in the public directory e.g. dirname/images. Default value : images

body_editor:xoopsdhtml

WYSIWYG editor will be enabled only for forums allowing HTML.

anonymous_name:guest

Anonymous Name. A pseudonym or alias is a fictitious name that a person or group assumes for a particular purpose, which differs from their original or true name. Default value: guest

icon_meanings:none|normal|unhappy|happy|lower it|raise it|report|question

The alt attribute provides alternative information explaining the meaning of forum icons. Specify each alt separated by pipe "|".

guest_vote_interval:86400

Enable Voting from guests. Allow votes from the same IP with required delay in seconds. Disable with value set to 0. Default value: 86400.

Comments Integration

The comments-integration feature of the d3forum module is particularly valuable for involving users in discussion forums. In addition to increasing usability, the integration makes commenting quick and easy.

This feature allows you to include a quick form and recent comments in any page and any blog. You can customize the design using the Components module or Template Management.

Before integrating into a module, for example Pico, create a category and a forum to manage module comments.
You can then modify the module preferences.

Create a category for "Comments"

  1. Browse to D3forum in your site's front end
  2. Click the action "New category"
  3. Enter a name e.g. "Comments"
  4. Enter a category description
  5. Click the button "Submit"

Create a forum for comment integration

  1. Browse to your category "Comments"
  2. Enter a name< e.g. "Pico's comments"
  3. Enter a forum description
  4. Select the category : "Comments"
  5. Forum weight : you can leave the default value "0"
  6. Format for comment-integration :
    Leave blank, the module's feature comment-integration will set this field automatically.
  7. Click the button "Submit"

Get the Forum ID for comment integration

The forum ID's can be found by browsing to the new forum "Pico's comments" URL.

.../modules/d3forum/index.php?forum_id=1

The ID is what comes after forum_id=, so in this case, it is 1.

Set the Forum ID in Module's preferences

Although this example uses the Pico module preferences, the steps in the comment integration process are the same for all modules.

Module Blocks

The module allows you to add blocks with forums, topics and posts in the different modules and pages of your site.

Each block has its own options and template, permissions and cache settings.

Block Forums

  • Specify categories. Set category's id separated by a comma. Note this specification is not applied recursively. Leave the fill blank for all categories, or `auto` for the current category.
  • Cache Time

Block Topics

  • Display items. Default value : 10
  • Display sie, sorted, marked
  • Set category's id separated by a comma. Note this specification is not applied recursively. Leave the fill blank for all categories, or `auto` for the current category.
  • Specify forums. Same as category specification.
  • Cache Time

Block Posts

  • Display items. Default value : 10
  • Display sie, sorted, marked
  • Set category's id separated by a comma. Note this specification is not applied recursively. Leave the fill blank for all categories, or `auto` for the current category.
  • Specify forums. Same as category specification.
  • Cache Time

Preferences

Module Preferences focus primarily on how the forums, forum topics, and comments are set on the site.

The D3 Forum module comes with several options set for a new forum.

Review the list of options and define your preferences.

If you're familiar with themes and templates, you can duplicate the template files of the module you want to customize.

You can install multiple instances of D3Forum using the X-Update Manager.
